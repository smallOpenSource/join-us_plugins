# monthly-report — joinus 월간 봉사활동 보고서

## ⚙️ 설정 (필수 — 플레이스홀더 해소)

이 스킬 본문의 `<OWNER>/<REPO>`·`<my-gh-login>`·`<git-author-name>`·`<project-root>` 등은 **범용화 플레이스홀더**다. 실행 전 반드시 실제 값으로 해소한다:

1. **설정 파일 로드**(KEY=value, 첫 발견 우선): `$JOINUS_CONFIG` → `./.join-us.env` → `~/.config/join-us/config.env`. `set -a; . <file>` 로 export 하거나 값을 읽어 치환한다. 템플릿 = `config/join-us.env.example`(`join-us config --init` 로 생성).
2. **설정이 없으면 사용자에게 1회 질문**해 값을 확보(인터랙션)하고 같은 세션 동안 재사용한다.

치환표: `<OWNER>/<REPO>`→`$JOINUS_REPO` · `<my-gh-login>`→`$JOINUS_GH_LOGIN`(구계정=`$JOINUS_GH_LOGIN_ALT`) · `<git-author-name>`→`$JOINUS_AUTHOR_NAME` · `<project-root>`→`$JOINUS_PROJECT_ROOT` · `<project-domain>`→`$JOINUS_WIKI_DOMAIN` · 팀원 제외 목록→`$JOINUS_TEAM_LOGINS` · `<plaintext-pw>`(알려진 평문 시크릿 스캔 패턴)→`$JOINUS_SECRET_PATTERNS`.

> ⚠️ 실값 설정 파일은 **비공개**(별도 관리) — 절대 커밋/게시하지 않는다. 공개본엔 `*.example`(플레이스홀더)만 포함된다. 본문의 "시크릿 0" 게이트는 그대로 유지한다.


## Purpose

joinus 프로젝트에서 **본인(<my-gh-login> = git author "<git-author-name>")** 의 한 달 작업을 모아, 팀 봉사활동 보고서 양식(`wiki_docs/volunteer_work_review_format.txt`)대로 `SR/monthly_summary/<YYYY-MM>.md` 에 작성한다. AI 실행시간이 아니라 **사람이 실제 들이는 worktime** 기준으로 시간을 산정한다.

## When to Use

- "월간 보고서 / 봉사활동 보고서 / monthly report 작성", `/monthly-report`, `/monthly-report 2026-05` 등.
- 매월 1회(양식이 "월간 단위 게시")인 반복 작업.

## ⛔ HARD GATES (어기면 중단)

1. **본인 작업만.** `<my-gh-login>`(=<git-author-name>) 의 PR/op 만. 팀원(다른 기여자들·dependabot 등) 작업 **오귀속 금지**.
2. **양식 5섹션** (제목블록·메타 불릿은 넣지 않음 — 양식에 없음). `활동 내용 요약` / `활동 결과물`(PR 링크) / `의견/후기` / `활동 일시` / `총 합산시간`. **`활동 결과물` 에 본인 PR 링크 필수**(2026-06 사용자 재확인 — 링크 누락 금지).
3. **시크릿 0.** AWS 인스턴스 ID(`i-...`)·계정번호(12자리)·DB 비번·bastion 호스트·`.env` 값 **절대 미기재**. 작성 후 grep 스크럽 필수.
4. **로컬 전용.** `SR/` 는 `.gitignore` → **커밋/PR/`change_log.md` 기록 금지**. 파일만 생성.
5. **시간 3곳 일치.** `활동 내용 요약` · `활동 일시` · `총 합산시간` 의 일별/총합이 **완전히 동일**해야 함(작성 후 검증).

## 산출물

- 경로: `<project-root>/SR/monthly_summary/<YYYY-MM>.md` (예: `2026-05.md`)
- 양식 기준: `wiki_docs/volunteer_work_review_format.txt`

## 섹션 구조 (정확히 5개, 이 순서)

1. `# 활동 내용 요약` — **날짜별 블록**: 헤더 `M월 D일 (요일) 12:00~20:00 ( 8시간 )` 한 줄, 이후 **활동·근거를 각각 `- ` 불릿**(활동 먼저, 근거 문장은 뒤 불릿). 엔트리 사이는 빈 줄(2026-05 파일은 U+200B 공백 줄을 spacer로 사용).
2. `# 활동 결과물` — **본인 PR 링크 목록**: `- {활동}: https://github.com/<OWNER>/<REPO>/pull/<N>`. PR 없는 산출물(내부 스킬 등)은 이름만 기재.
3. `# 의견/후기` — 산문 회고. 6영역(개발·디버깅·트러블슈팅·스킬·온보딩·개발환경) 포괄.
4. `# 활동 일시` — **날짜별 1줄**(근거 없음): `M월 D일 (요일) 12:00~20:00 / 8시간 / 활동A + 활동B + …`. 상단에 "토·일 기재" note.
5. `# 총 합산시간` — 총합 `N시간` 한 줄, 이후 날짜별 `YYYY-MM-DD (HH:MM-HH:MM) N시간`(**ISO 날짜·시각은 `-` 하이픈·불릿/요일 없음**).

## 줄 형식 규칙 (섹션별로 다름 — 2026-06-25 확정)

- **`활동 내용 요약`** = 날짜별 **블록**: 헤더 `M월 D일 (요일) 12:00~20:00 ( 8시간 )`(공백 포함 `( 8시간 )`, 시각은 `~`) + 활동/근거를 **각 줄 `- ` 불릿**(활동 먼저, 근거 문장 뒤). 엔트리 사이 빈 줄(U+200B 공백 줄 spacer).
- **`활동 일시`** = 날짜별 **슬래시 1줄**(근거 없음): `M월 D일 (요일) 12:00~20:00 / 8시간 / 활동A + 활동B + …`(활동은 ` + `로 연결).
- **`총 합산시간`** = `N시간` + 날짜별 `YYYY-MM-DD (12:00-20:00) 8시간`(ISO·`-`하이픈·불릿/요일 없음).
- 공통: 토·일만, 매일 `12:00~20:00`(8시간), 요일 `(토)/(일)` 표기.

## 출력 템플릿 (사용자 확정 형태 — 2026-06-25)

아래 형태를 **그대로** 따른다(제목블록 없음, 5섹션). 섹션별 양식이 **다름**: `활동 내용 요약`=헤더+`- ` 불릿 블록, `활동 일시`=슬래시 1줄, `총 합산시간`=ISO 날짜. 날짜는 그 달 값으로 교체. (엔트리 사이 빈 줄의 `​`은 U+200B 공백 줄 spacer.)

```markdown
# 활동 내용 요약

6월 6일 (토) 12:00~20:00 ( 8시간 )
- 활동A
- 활동B
- {근거 문장1}
- {근거 문장2}

​

6월 7일 (일) 12:00~20:00 ( 8시간 )
- 활동C
- …
- {근거 문장}


# 활동 결과물

- {활동명}: https://github.com/<OWNER>/<REPO>/pull/<N>
- {활동명}: https://github.com/<OWNER>/<REPO>/pull/<N> , https://github.com/<OWNER>/<REPO>/pull/<M>
- 개발 보조 도구(내부, 저장소 외부): {스킬명 등 PR 없는 산출물은 이름만}


# 의견/후기

{산문 — 문장별 줄바꿈; 개발·디버깅·트러블슈팅·스킬·온보딩·개발환경 6영역 포괄}


# 활동 일시

> 봉사 일시는 토·일로 기재함 (매일 12:00~20:00, 각 8시간; 주중 수행분을 인접 주말에 합산). 날짜별로 행 구분.

6월 6일 (토) 12:00~20:00 / 8시간 / 활동A + 활동B
6월 7일 (일) 12:00~20:00 / 8시간 / 활동C + …
…(활동 내용 요약과 동일 활동, 근거 제외)


# 총 합산시간

64시간

2026-06-06 (12:00-20:00) 8시간
2026-06-07 (12:00-20:00) 8시간
…(토·일 날짜별, ISO 날짜)
```

> 하루 **8시간 고정**(매일 12:00~20:00), **토·일만**. 월 총량 = (토·일 봉사일 수) × 8h. **양식**: `활동 내용 요약`=헤더+불릿, `활동 일시`=슬래시, `총 합산시간`=ISO. 날짜는 그 달 작업이 몰린 주의 토/일로 교체.

## 날짜 (토·일 기재)

- 봉사일은 **토·일(주말)로만** 기재. 평일 수행분을 인접 주말에 합산.
- 작성 전 그 달의 토/일 날짜를 `date`로 계산한다. 실제 작업이 몰린 주의 토/일을 사용.
- ⚠️ 미래 날짜는 보통 미기재(완료분만). **사용자 명시 요청 시에만** 예외(예: 2026-06-25 요청으로 6/27·28 포함).

## 시간 (인간 worktime)

- AI 실행시간이 아니라 사람이 조사·재현·디버깅·검증에 들이는 시간.
- **매일 12:00~20:00, 하루 8시간 고정(2026-06-25 확정)**: 요일 무관 동일. 토·일만 봉사일.
- 월별 총량 = (토·일 봉사일 수) × 8h. 실측: 2026-05 = 4일(5/23·24·30·31) = 32h, 2026-06 = 8일(6/6·7·13·14·20·21·27·28) = 64h. (구 이력: 16h×주말08-24 → 금토일월×8h[금월16-24·토일12-20] → 토·일×8h/12-20.)

## 데이터 출처 (본인 작업 식별)

- `wiki_docs/change_log.md`: `작성자 = <my-gh-login>` 행 + `op`(본인 운영조치) 행. → 여기서 PR 번호 수집(`활동 결과물` 링크 소스).
- `git log --since=<월초> --until=<다음달초> --author='<git-author-name>'` (또는 <my-gh-login>).
- 설치된 스킬 디렉터리(예: `~/.claude/skills/` · `~/.codex/skills/join-us-*`): 그 달 mtime 인 **본인 빌드 스킬**(init-join-us·pr-join-us·review-before-pr-join-us 등). `stat -c '%y' .../SKILL.md` 로 월 확인.
- `SR/reports/`: 운영조치 상세(보고서 본문).

## Workflow

1. **대상 월 확정** — 인자(`/monthly-report 2026-05`) 또는 질문. 그 달 토/일 날짜 계산(`date`).
2. **본인 작업 수집** — change_log author 필터 + `git log --author` + 스킬 mtime + SR/reports. 팀원분 제외(HARD GATE 1). **PR 번호도 함께 수집**.
3. **하루 총량 확인** — 사용자에게 시간 기준 확인(인간 worktime). 미응답 시 직전 달 기준 제안.
4. **테마 매핑** — 작업을 6영역 테마로 묶어 토/일 날짜에 배치(날짜별 1줄로 합칠 활동 목록 확정; 매일 12:00~20:00).
5. **작성** — 5섹션 보고서를 `SR/monthly_summary/<YYYY-MM>.md` 에 작성(**요약=헤더+`- ` 불릿, 일시=슬래시 1줄, 총합산=ISO 날짜**; 결과물=PR 링크 목록).
6. **검증** — 아래 명령으로 ①섹션 5개 ②PR 링크 존재 ③매일 12:00~20:00·각 8h ④3곳 시간합 일치 ⑤토·일 날짜만(요일 표기) ⑥시크릿 0 확인.
7. **보고** — 요약 + 검증 결과. (커밋/PR/change_log **하지 않음**.)

## 검증 명령 (작성 후 필수)

```bash
F=<project-root>/SR/monthly_summary/<YYYY-MM>.md
grep -cE '^# (활동 내용 요약|활동 결과물|의견/후기|활동 일시|총 합산시간)$' "$F"   # = 5
grep -oE 'pull/[0-9]+' "$F" | sort -u | wc -l                                  # ≥ 1 (본인 PR 링크 존재)
grep -cE 'i-[0-9a-f]{8,}|\b[0-9]{12}\b|password\s*[:=]|ec2-[0-9-]+\.|ip-10-' "$F"  # = 0
grep -oE '[0-9]+월 [0-9]+일 \((토|일)\)' "$F" | wc -l   # 모든 날짜행이 토·일 요일표기인지(개수=날짜행수)
# python 으로: ①매일 12:00~20:00·각 8시간 ②활동내용요약(헤더)/활동일시(슬래시)/총합산(ISO) 3곳의 날짜집합·일수 동일 ③합계=토·일 봉사일수×8h ④총합산 줄=`YYYY-MM-DD (12:00-20:00) 8시간` ISO 형식
```

## Success Criteria

- 정확히 5섹션(`활동 결과물` 에 본인 PR 링크 포함), **요약=헤더+`- ` 불릿·일시=슬래시·총합산=ISO 날짜**, **토·일 날짜만·매일 12:00~20:00·각 8h**, **3곳 날짜/시간 일치**, 시크릿 0.
- 본인 작업만, 로컬 전용(미커밋), 양식 충실.

## Pitfalls

- `활동 결과물`(PR 링크) **누락** → 양식 필수. change_log 의 본인 PR 번호로 링크 채울 것. (단 제목블록·메타 불릿은 넣지 않음.)
- 토·일 아닌 날 기재, 또는 12:00~20:00(8h) 아닌 시각 → 오답. **매일 `12:00~20:00`, 토·일만**.
- 팀원 PR 오귀속.
- 섹션 양식 혼동 → `활동 내용 요약`=헤더+`- ` 불릿(활동·근거 각 줄), `활동 일시`=슬래시 1줄(근거 제외), `총 합산시간`=`YYYY-MM-DD (12:00-20:00) 8시간` ISO. 서로 바꿔 쓰지 말 것.
- `SR/` 산출물을 커밋하거나 change_log 에 기록 → 금지(로컬 전용).
